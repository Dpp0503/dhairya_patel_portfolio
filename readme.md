## 🌟 Featured Projects

### 🏥 [CareNexus](https://github.com/Dpp0503/CareNexus) | Hospital Management System
**Core Problem:** Healthcare facilities often struggle with fragmented systems for managing patient records, billing, and doctor schedules, leading to inefficiencies.
**Solution:** A comprehensive, unified 'nexus' that centralizes patient history, digital prescriptions, role-based access, and automated billing through a clean REST API.
**Architecture & Tech Stack:**
- **Backend:** Java 25, Spring Boot 4.0.2, PostgreSQL, Hibernate/JPA, Spring Security, JWT, Razorpay Integration.
- **Frontend:** React.js, Tailwind CSS, Redux Toolkit.
- **Design Choices:** Uses a layered MVC architecture. Authentication flows through JWTs, while complex relationships (e.g., Doctors -> Departments -> Appointments -> Patients) are strictly modeled in PostgreSQL. The frontend uses dynamic state management via Redux to handle real-time appointment updates and secure payment sequences via Razorpay webhooks.

### 🎬 [StreamSphere](https://github.com/Dpp0503/OTT_FULL_STACK) | Premium OTT Streaming Platform
**Core Problem:** Building a scalable, secure media streaming platform that supports personalized experiences (multiple profiles) and handles rich media delivery efficiently.
**Solution:** A feature-rich MERN stack platform featuring secure subscription management, isolated multi-profile data (watch history, watchlists), and an immersive dark-themed UI.
**Architecture & Tech Stack:**
- **Backend:** Node.js, Express.js, MongoDB (Mongoose ODM).
- **Frontend:** React, Tailwind CSS, Vite, Redux Toolkit, Redux Persist.
- **Infrastructure & Media:** Docker (Containerization), Cloudinary (Optimized media delivery).
- **Design Choices:** Fully containerized using Docker Compose for seamless deployment. Complex NoSQL relationships model Series -> Seasons -> Episodes. The frontend prioritizes a "Content-First" dark aesthetic, fully responsive from mobile to 4K displays, with micro-interactions and skeleton loaders for premium UX.

### 🪐 [ExoFinder](https://github.com/Dpp0503/Exofinder) | NASA Space App Challenge 2025
**Core Problem:** Analyzing massive amounts of astronomical data from Kepler, K2, and TESS missions to find subtle planetary transit signals is highly computationally intensive and difficult to classify manually.
**Solution:** An advanced AI-powered exoplanet discovery system that analyzes light curves directly to identify, classify, and assess the habitability of potential exoplanets.
**Architecture & Tech Stack:**
- **AI/ML:** Scikit-learn, TensorFlow/Keras, CNNs (Direct light curve analysis), Hybrid Ensemble Models.
- **Backend:** Python, Flask API, Pandas, NumPy.
- **Frontend:** HTML, CSS (Bootstrap), JavaScript, Chart.js.
- **Design Choices:** Implements a multi-dataset integration pipeline. Uses adaptive feature engineering and a novel CNN architecture designed specifically for raw light curve analysis, eliminating the need for extensive pre-processing. Includes a habitability calculator factoring in stellar temperature, orbital distance, and planetary size.
